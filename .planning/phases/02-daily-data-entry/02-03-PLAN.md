# Plan 02-03: Cost Entry Section + Unsaved Changes Guard

<objective>
Add the expandable cost entry section (D-05) and unsaved changes confirmation dialog (D-09) to the daily entry form. Cost entry is optional — collapsed by default, available when the owner wants to log exact costs. The unsaved changes guard prevents accidental data loss when navigating dates.
</objective>

<must_haves>
- Expandable cost section below revenue sections, collapsed by default
- One cost row per category (all 6 non-party categories) with € cost input + Exact/Estimated toggle
- Margin hint showing estimated cost when revenue exists but no exact cost entered
- Cost data included in the PUT payload when saving
- Unsaved changes dialog when navigating away with dirty form
- Dialog offers "Discard" and "Save & continue" actions
</must_haves>

   ```typescript
   const [initialFormSnapshot, setInitialFormSnapshot] = useState<string>('')
   // After loading data, JSON.stringify the initial form state → setInitialFormSnapshot
   // isDirty = JSON.stringify(currentFormState) !== initialFormSnapshot
   ```

3. **Pending navigation:** When user tries to navigate (prev/next/today/picker) and `isDirty` is true:
   - Store the intended target date in `pendingDate: Date | null`
   - Return `pendingDate` and `isDirty` so the page can show the dialog

   - Provide `confirmSave()` → saves current data, then navigates to pendingDate on success

4. Update `goToPrevDay`, `goToNextDay`, `goToToday`, `setSelectedDate` to check dirty state before navigating. If dirty, set pendingDate instead of navigating directly.

Export new values: `isDirty`, `pendingDate`, `confirmDiscard`, `confirmSave`, `cancelNavigation`, cost state + setters.
