# Plan 02-01: API Routes + shadcn Setup + Client Routing

<objective>
Build the server-side API for daily entry CRUD, initialize shadcn/ui in the client, and set up React Router so the daily entry page is the default view. This is the foundation that Wave 2 plans build on.
</objective>

<must_haves>
- GET /api/daily-entries/:date returns all entries + kids count for a date
- PUT /api/daily-entries/:date upserts revenue entries + kids count for a date
- shadcn/ui initialized in client with brand colors
- React Router installed with / route pointing to DailyEntryPage
- Date-fns installed for date formatting
</must_haves>

<action>
Create `server/src/routes/daily-entries.ts` with two endpoints:

**GET /** — Query param `date` (YYYY-MM-DD format).
Returns JSON:
```json
{
  "date": "2026-03-27",
  "entries": [
    { "categoryId": 1, "categoryName": "toys", "displayName": "Toys", "businessArea": "retail", "revenue": 150.50, "costAmount": null, "costBasis": "estimated", "costMarginPct": 0.55 }
  ],
  "kidsCount": 12,
  "lastSaved": "2026-03-27T14:32:00.000Z"
}
```

Query logic:
- Left join `categories` (active=true, ordered by sortOrder) with `dailyEntries` filtered by date
- Separately query `kidsEntries` for the date
- `lastSaved` = max of updatedAt across all dailyEntries + kidsEntries for that date, or null
- Categories with no entry return `revenue: null, costAmount: null` (D-08: empty = not entered)

**PUT /** — Body validated with Zod:
```typescript
const saveSchema = z.object({
  date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
  entries: z.array(z.object({
    categoryId: z.number().int().positive(),
    revenue: z.number().min(0),
    costAmount: z.number().min(0).nullable().optional(),
    costBasis: z.enum(['exact', 'estimated']).optional().default('estimated'),
  })),
  kidsCount: z.number().int().min(0).nullable().optional(),
})
```

Save logic:
- For each entry in `entries`: upsert into `dailyEntries` using `onConflictDoUpdate` on the unique index (entryDate, categoryId). Set `updatedAt` to current datetime.
- Only save entries where revenue is provided (skip entries with null/undefined revenue — D-08)
- If `kidsCount` is a number (including 0): upsert into `kidsEntries` with `onConflictDoUpdate` on entryDate unique constraint
- If `kidsCount` is null/undefined: do not touch kidsEntries
- Return `{ success: true, date, savedAt: new Date().toISOString() }`

Use drizzle-orm imports: `eq`, `and`, `sql`, `max` from `drizzle-orm`. Use `db.insert().values().onConflictDoUpdate()` for upserts.
</action>

- File exports default Hono route
