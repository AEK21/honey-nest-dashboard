## Task Details

### Task 1: KidsTrendChart
Recharts `LineChart` or `AreaChart`:
- Single line: Dusty Mint `#BFD8D2` with light gradient fill
- X-axis: dates (last 30 days), formatted as "Mar 1", "Mar 2"
- Y-axis: integer count
- Smaller than revenue chart — height ~180px
- Tooltip: date + "X kids"
- Card wrapper with title "Kids Entries (30 days)"

This chart is independent of the area filter (kids are always playroom).

### Task 2: ProfitBreakdown
A compact card showing per-category profit detail:
- Category name | Revenue | Cost | Profit | Basis indicator (dot: green for exact, amber for estimated)
- Sorted by profit descending
- Totals row at bottom
- Uses data from the categories endpoint (already has revenue + cost + profit)
- Responds to area filter

Styled as a compact table with `text-[13px]`, alternating row bg (`#1E1B18` / `#252220`), header `#9A8E82`.

### Task 3: Empty States
For each dashboard section, when no data exists:
- KPI cards: show "---" for value, no delta badge
- Revenue trend: "No entries yet for [Month]" centered in chart area
- Category breakdown: "Enter some data to see breakdowns"
- Kids trend: "No kids entries in the last 30 days"
- Profit breakdown: "No data for this period"

All empty states use `text-[#7A6F63]` with a subtle icon or illustration if appropriate.

### Task 4: Dashboard Integration
Final page layout (top to bottom):
```
[Page title: "Dashboard" + month subtitle]
[BusinessAreaFilter]
[KPI Cards row: Today | MTD Revenue | Gross Profit]
[RevenueTrendChart]
[CategoryBreakdownChart + ProfitBreakdown — side by side on wider screens, stacked on narrow]
[KidsTrendChart]
```

### Task 5: Polish
- Consistent spacing: `space-y-5` between sections
- Bottom padding for nav bar overlap (pb-24)
- Subtle fade-in animation on data load (opacity transition like entry page)
- Ensure charts resize correctly on mobile widths (320px+)
- Verify all dark theme colors are consistent with Phase 2
