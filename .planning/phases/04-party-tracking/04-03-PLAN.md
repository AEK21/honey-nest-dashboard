# Plan 04-03: Dashboard Integration

**Phase:** 04 — Party Tracking
**Wave:** 3 of 3
**Goal:** Wire party revenue into the dashboard — enable Parties filter, add party summary widget, include party data in trend chart.

## Tasks

| # | Task | Key Files |
|---|------|-----------|
| 1 | Enable "Parties" pill in BusinessAreaFilter | `client/src/features/dashboard/components/BusinessAreaFilter.tsx`, `client/src/features/dashboard/hooks/useDashboard.ts` |
| 2 | Create PartyRevenueSummary widget | `client/src/features/dashboard/components/PartyRevenueSummary.tsx` |
| 3 | Add usePartyDashboard hook | `client/src/features/dashboard/hooks/usePartyDashboard.ts` |
| 4 | Integrate widget into DashboardPage | `client/src/features/dashboard/DashboardPage.tsx` |

## Task Details

### Task 1: Enable Parties Filter
- Add `'parties'` to `AreaFilter` type
- Add `{ value: 'parties', label: 'Parties' }` to options (remove disabled)
- Server already handles `parties` in areaFilter from Wave 1

### Task 2: PartyRevenueSummary
Card showing (for completed parties in selected month):
- Party count
- Package revenue vs. add-on revenue (stacked or side-by-side)
- Total party revenue
- Avg revenue per party
- Shown only when area is 'all' or 'parties'

### Task 3: usePartyDashboard Hook
- Fetches `/api/dashboard/parties?month=X`
- Returns party summary data

### Task 4: DashboardPage Integration
- Add PartyRevenueSummary below KPI cards (or below category chart)
- Party revenue appears in existing trend chart when area = 'all' (via server aggregation)
- Party categories appear in category breakdown (party_packages + party_addons categories)
