# Plan 04-01: Schema Migration + Party CRUD API

**Phase:** 04 — Party Tracking
**Wave:** 1 of 3
**Goal:** Expand the parties schema, create CRUD API, and add dashboard aggregation endpoint.

## Tasks

| # | Task | Key Files |

## Task Details

### Task 1: Schema Update

Add to `parties` table:
```
party_time     text        nullable
customer_name  text        not null
child_name     text        nullable
child_age      integer     nullable
kids_count     integer     nullable
adults_count   integer     nullable
deposit_amount real        nullable
status         text        not null  default 'booked'
event_type     text        nullable
```

### Task 2: Migration

Run `npx drizzle-kit generate` then apply. Since this is dev with no production data, can also use `drizzle-kit push` if simpler.

### Task 3: Parties CRUD

`GET /api/parties?month=YYYY-MM` — list parties for month, each with nested addons array, ordered by date+time. Include computed `addonsTotal` and `totalRevenue`.

`GET /api/parties/:id` — single party with addons.

`POST /api/parties` — Zod-validated. Creates party + addons in one request. Body:
```json
{
  "partyDate": "2026-04-15",
  "partyTime": "15:00",
  "customerName": "Maria K.",
  "childName": "Sofia",
  "childAge": 6,
  "kidsCount": 12,
  "adultsCount": 4,
  "packageName": "Premium",
  "packagePrice": 180,
  "depositAmount": 50,
  "status": "booked",
  "eventType": "birthday",
  "notes": "Allergic to nuts",
  "addons": [
    { "addonName": "Extra pizza tray", "addonPrice": 25, "category": "food" },
    { "addonName": "Balloon arch", "addonPrice": 35, "category": "decorations" }
  ]
}
```

`PUT /api/parties/:id` — full replace of party + addons (delete old addons, insert new).

`DELETE /api/parties/:id` — cascade deletes addons.

### Task 4: Dashboard Aggregation

`GET /api/dashboard/parties?month=YYYY-MM`

Returns (only status = 'completed'):
```json
{
  "partyCount": 8,
  "packageRevenue": 1440,
  "addonRevenue": 520,
  "totalRevenue": 1960,
  "avgPerParty": 245
}
```

Also add `parties` to the `validAreas` array and `areaFilter` function so existing summary/trend/categories endpoints can include party data when area = 'all' or area = 'parties'.

### Task 5: Mount Route

Add `import partiesRoute from './routes/parties'` and mount at `/api/parties`.
