# Plan 04-02: Party Entry UI

**Phase:** 04 — Party Tracking
**Wave:** 2 of 3
**Goal:** Build the party list page and party form — the owner can create, view, edit, and delete parties with add-ons.

## Tasks

| # | Task | Key Files |
|---|------|-----------|
| 1 | Create useParties hook (list query + mutations) | `client/src/features/parties/hooks/useParties.ts` |
| 2 | Create PartyCard component | `client/src/features/parties/components/PartyCard.tsx` |
| 3 | Create PartyForm component (modal/sheet) | `client/src/features/parties/components/PartyForm.tsx` |
| 4 | Create PartyListPage | `client/src/features/parties/PartyListPage.tsx` |
| 5 | Add /parties route and update BottomNav (3 tabs) | `client/src/App.tsx`, `client/src/components/BottomNav.tsx` |

## Task Details

### Task 1: useParties Hook
- Fetches `/api/parties?month=YYYY-MM`
- Month navigation (reuse pattern from useDashboard)
- Create, update, delete mutations
- Invalidates query on success

### Task 2: PartyCard
Compact card showing:
- Date + time (left), status badge (right)
- Customer name, child name + age
- Package name + price
- Add-on count + total
- Total revenue (package + addons) — prominent
- Tap to edit, long-press or icon to delete

Status badge colors:
- inquiry: `#9A8E82` muted
- booked: `#E7C76A` honey yellow
- completed: `#6ECF8E` green
- cancelled: `#D4564E` red

### Task 3: PartyForm
Full-screen modal or slide-up sheet:
- Date picker + optional time input
- Customer name (required)
- Child name + age (optional row)
- Kids count + adults count (side by side)
- Event type (text input, placeholder examples)
- Package name (text input) + package price (euro input)
- Deposit amount (euro input, optional)
- Status selector (4 pills)
- Notes (textarea)
- **Add-ons section:** dynamic rows, each with name + price + category. "Add add-on" button. Remove button per row.
- Save / Cancel buttons

Dark themed, consistent with daily entry form styling.

### Task 4: PartyListPage
- Month navigation header (same pattern as dashboard)
- Party cards grouped by date
- Floating "+" button to add new party
- Empty state: "No parties this month"
- Status filter pills: All / Inquiry / Booked / Completed / Cancelled (optional — can defer)

### Task 5: Routing + Nav
- Add `/parties` route
- BottomNav: 3 tabs — Entry (PenLine) / Parties (Cake) / Dashboard (BarChart3)
