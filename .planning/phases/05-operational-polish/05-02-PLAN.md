# Plan 05-02: PWA & Mobile Polish

**Phase:** 05 — Operational Polish
**Wave:** 2 of 3
**Goal:** Make the app installable on mobile (home screen icon) and feel native. Add proper meta tags, theme color, and a web app manifest.

## Tasks

| # | Task | Key Files |
|---|------|-----------|
| 1 | Create PWA manifest.json | `client/public/manifest.json` |
| 2 | Create app icons (SVG + PNG fallbacks) | `client/public/icons/` |
| 3 | Update index.html with meta tags + manifest link | `client/index.html` |
| 4 | Add dynamic page titles per route | `client/src/App.tsx` or hook |
| 5 | Add apple-touch-icon and mobile status bar styling | `client/index.html` |

## Task Details

### Task 1: Web manifest
```json
{
  "name": "Honey Nest Dashboard",
  "short_name": "Honey Nest",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#1C1917",
  "theme_color": "#E7C76A",
  "icons": [...]
}
```

### Task 2: App icons
- Create a simple SVG icon (honeycomb/hexagon motif in Honey Yellow on Dark Brown)
- Generate 192x192 and 512x512 PNG versions
- Since we can't generate raster images in code, use an inline SVG as the icon source and reference it

### Task 3: Meta tags
- `<meta name="theme-color" content="#1C1917">` — matches dark background
- `<meta name="apple-mobile-web-app-capable" content="yes">`
- `<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">`
- `<link rel="manifest" href="/manifest.json">`

### Task 4: Dynamic page titles
- Entry page: "Honey Nest — Entry"
- Parties: "Honey Nest — Parties"
- Dashboard: "Honey Nest — Dashboard"
- Use a simple `useEffect` + `document.title` per page, or a shared hook

### Task 5: Apple touch icon
- `<link rel="apple-touch-icon" href="/icons/icon-192.png">`
