# Plan 05-01: Data Export & Safety Net

**Phase:** 05 — Operational Polish
**Wave:** 1 of 3
**Goal:** Give the owner a way to get data out of the system. CSV export for daily entries and parties. This is the #1 trust builder — "I can always get my data."

## Tasks

| # | Task | Key Files |
|---|------|-----------|
| 1 | Add GET `/api/export/entries?month=YYYY-MM` endpoint | `server/src/routes/export.ts` |
| 2 | Add GET `/api/export/parties?month=YYYY-MM` endpoint | `server/src/routes/export.ts` |
| 3 | Mount export route in server index | `server/src/index.ts` |
| 4 | Add export buttons to Dashboard page (entries CSV) and Party list (parties CSV) | `DashboardPage.tsx`, `PartyListPage.tsx` |

## Task Details

### Task 1: Export entries CSV
- GET `/api/export/entries?month=2026-03`
- Returns CSV with headers: `Date, Category, Business Area, Revenue, Cost, Cost Basis`
- One row per daily_entry for the month, joined with category name
- Content-Type: `text/csv`, Content-Disposition: `attachment; filename="entries-2026-03.csv"`
- If no month param, default to current month

### Task 2: Export parties CSV
- GET `/api/export/parties?month=2026-03`
- Returns CSV with headers: `Date, Time, Customer, Child, Package, Package Price, Add-ons Total, Total Revenue, Deposit, Status`
- One row per party, add-ons summed into a total column
- Content-Type: `text/csv`, Content-Disposition: `attachment; filename="parties-2026-03.csv"`

### Task 3: Mount route
- `app.route('/api/export', exportRoute)` in server index

### Task 4: Export buttons
- Dashboard: small "Export CSV" text button in the header area, exports entries for current month
- Party list: small "Export CSV" text button in the header, exports parties for current month
- Both trigger a download via `window.open('/api/export/...')`
